# Contributing to ACDP (Agent Coordination Protocol for Development)

First off, thank you for considering contributing to ACDP! 

Building a robust protocol requires architectural discipline. We prioritize **concepts over code** and **solid foundations over speed**.

## The Golden Rule: Issue-First 🚧

**Do not submit surprise Pull Requests.** 

If you are proposing a significant change, protocol modification, or structural enhancement, we strictly follow an **Issue-First / Request for Comments (RFC)** methodology. 

1. **Discuss the Problem**: Open an Issue to discuss the problem you want to solve. Code without context or understanding is the wrong solution.
2. **Propose the Solution (RFC)**: Outline your proposed architectural approach in the Issue. Explain *why* it's the right approach and outline any trade-offs.
3. **Reach Consensus**: Wait for a project maintainer to review and approve the technical approach.
4. **Execute**: Once the approach is approved, you may submit your Pull Request referring to the approved Issue.

### Why do we do this?
Because fixing the wrong architectural approach during a PR review is painful for everyone. A solid foundation must be laid *before* construction begins. Real learning and good design take effort and time.

## Pull Request Process

1. Ensure your PR is linked to an approved Issue.
2. Use **Conventional Commits** for your commit messages.
3. Ensure any relevant schemas (e.g., `messages.schema.json`) and documentation files are updated to reflect your changes.
4. Do not include AI attribution in commits. 
5. Wait for the maintainers to review. We will evaluate the implementation against the agreed-upon RFC.

Thank you for helping us build a more solid standard for AI agent coordination.
